Cracker无法连接

  1. 首先检查Cracker是否正常工作，Cracker的电源指示灯是否亮起、显示屏是否已经显示IP地址？

    • 电源指示灯不正确：请检查是否插入电源。或者您使用POE供电方式，请检查您的交换机是否支持POE供电，如过您的设备不支持POE供电请改用电源供电。
    • 显示屏不显示：请检查FPGA核心板是否正确插入，是否有连接不牢固。
    • 电源指示灯和显示屏无异常：继续后续的检查。

  2. 检查Cracker的显示屏上的IP地址，是否与您上位机中配置的IP地址一致？

    • 不一致：修改上位机中目标Cracker的地址为Cracker显示屏中的地址。
    • 一致：继续后续的检查

  3. 通过执行ping命令检查上位机主机与Cracker的通信是否正常，正常通信的效果如下：

    • 不能正常ping通：
      • 检查Cracker网卡指示灯是否有闪烁，如果不闪烁请检查对端网络设备是否开启或端口已启用或网线是否有异常。
      • 如果您的上位机电脑和Cracker通过网线直连的方式进行连接的，请检查系统配置的IP地址是否与Cracker的IP地址是否在同一网段，并且需要是不同的IP地址， 例如Cracker出厂的默认IP地址是192.168.0.10/24，则您的电脑可以配置为192.168.0.11/24。
      • 如果您的上位机电脑和Cracker是通过已有的网络进行连接，请检查Cracker的IP地址是否允许在该网络中通信，如果需要修改IP地址，请参看手册中IP地址修改章节进行需改。
    • 能ping通：
      • 如果您是通过已有的网络进行连接，请检查网络中的防火墙是否放行了9761和9760端口。

如果以上检查都没有问题，但依然无法连接到Cracker请联系我们。
